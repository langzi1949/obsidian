
C++的类继承的逻辑和Java几乎一样，所以我这边只会给出相关的区别和需要注意的细节


基类就是一个普通的类，按照正常的语法逻辑写就行

# 1. 继承的基本语法

C++继承的基本语法是在派生类声明时，使用冒号`:`后跟上**继承方式** + **基类名称**

```cpp
class 派生类名 : [继承方式] 基类名 {
    // 派生类特有的成员
};


// 1. 基类 (Base Class)
class Animal {
public:
    void eat() { /* ... */ }
protected:
    int age;
private:
    int id; // 私有成员，外部不可访问
};

// 2. 派生类 (Derived Class) 使用 public 继承
class Dog : public Animal {
public:
    void bark() { /* ... */ }

    void setAge(int a) {
        age = a; // 可以直接访问基类的 protected 成员
        // id = 10; // 错误：不能访问基类的 private 成员
    }
};

// 3. 使用派生类
int main() {
    Dog myDog;
    myDog.eat(); // 派生类继承了基类的 public 方法
    myDog.bark(); 
    // myDog.age = 5; // 错误：age 是 protected
    return 0;
}
```


# 2. 什么是继承方式？

C++提供了3种继承方式，它们决定了基类成员在子类中的**最小访问权限**

| 继承方式      | 含义           | 对基类成员的影响                                           | 作用                     |
| --------- | ------------ | -------------------------------------------------- | ---------------------- |
| public    | ==**最常用的**== | 基类中的`public`仍为`public`<br>`protected`仍为`protected` | 实现`is-a`关系，保持基类接口不变    |
| protected | 很少用          | 基类中的`public`和`protected`都变成了`protected`            | 基类的成员只能被子类访问           |
| private   | 很罕见          | 基类的`public`和`protected`都变成了`private`               | 基类的成员只能在派生类中内部使用，不对外暴露 |
所以，<span class="b">核心原则：继承不会改变基类成员在基类内部的访问权限，它只改变基类成员在派生类和派生类之外的访问权限</span>

> [!danger]
> 需要说明的是，如果缺失继承方式，那么默认的继承方式就是`private`


# 3. 派生子类
需要注意的是，在C++中是多继承，所以没有`super`的语法，如果是子类中需要初始化基类怎么办？
可以使用初始化列表的方式进行初始化基类， 如果不调用基类的构造函数，程序会使用基类的默认构造函数。

```cpp
RatedPlayer::RatedPlayer(unsigned int r, const string &fn,
                         const string &ln, bool ht) : TableTennisPlayer(fn, ln, ht)
{
    rating = r;
}
```

其中`TableTennisPlayer(fn, ln, ht)`就是调用基类的构造函数，通过初始化列表的方式。

在继承中，构造函数和析构函数的执行顺序是固定的：
1. 构造时：先调用基类的构造函数，再调用派生类的构造函数
	1. 派生类必须负责初始化基类的不分，可以通过初始化列表显式调用基类的构造函数
2. 析构时：先调用派生类的析构函数，再调用基类的析构函数
	1. <span class="r">重要：</span> 如果要实现多态，基类的析构函数必须声明为`virtual`虚析构函数，以防止内存泄露


> [!note]
> 补充一个知识点
> 在C++中没有`private`私有类或者`protected`保护类，这个和Java不一样

# 4. 多态的实现

C++的多态，是通过`virtual`虚函数来实现的，在C++标准中，如果不把基类的方法设为`virtual`，就无法实现运行时多态。

## 4.1 虚函数
在C++中，多态性是通过**虚函数**和**基类指针/引用**来实现的

1. 在基类方法声明前加上`virtual` 关键字。 告诉编译器，这个函数的调用地址应该由运行时通过对象的虚函数表来确定
2. 必须通过基类的指针或者引用来调用才能实现多态。 只有这样才会去查询对象的实际类型，从而实现动态绑定


```cpp
class Base {
public:
    // 非虚函数 (Non-virtual)
    void show() { std::cout << "Base::show()" << std::endl; } 
};

class Derived : public Base {
public:
    // 覆盖了 Base::show()，但它是非虚的
    void show() { std::cout << "Derived::show()" << std::endl; } 
};

int main() {
    Derived d;
    Base* ptr = &d; 

    ptr->show(); // 关键点：调用哪个函数？

    // 结果输出：Base::show()
}
```
以上就是多态失败了，因为这个是非虚函数，导致在编译器在编译的时候就进行了静态绑定。

```cpp
class Base {
public:
    // 虚函数 (Virtual)
    virtual void show() { std::cout << "Base::show()" << std::endl; } 
};
// ... (Derived 类定义不变)

int main() {
    Derived d;
    Base* ptr = &d; 

    ptr->show(); // 关键点：调用哪个函数？

    // 结果输出：Derived::show()
}
```
只有这样才可以。
再一次补充上面的一个知识点

```cpp
Base &pt = Derived();  //  ❌❌❌❌❌
// 上面的代码是错误的，涉及到我们之前讨论的左值和右值的逻辑
const Base &pt = Derived();  // ✅✅✅✅✅✅
```

## 4.2 虚析构函数

在多态的基础上，我们要讨论一下虚析构函数？首先，顾名思义，虚析构函数就是析构函数上加上`virtual`。

那么为什么需要虚析构函数？

核心原则： **防止内存泄露**

如果通过基类指针删除派生类对象，基类的析构函数不是虚函数的话，就会导致运行时多态失败，进而导致内存泄露和未定义行为

```cpp
class Base {
public:
    ~Base() { std::cout << "Base 析构函数调用" << std::endl; } // ❌ 非虚析构函数
};

class Derived : public Base {
private:
    int* data;
public:
    Derived() { data = new int[10]; } // 派生类分配了资源
    ~Derived() { 
        delete[] data; // 派生类释放资源的关键步骤
        std::cout << "Derived 析构函数调用" << std::endl; 
    }
};

int main() {
    Base* ptr = new Derived(); // 基类指针指向派生类对象
    
    delete ptr; // 关键操作

    // 实际输出：
    // Base 析构函数调用
    
    // 结果：
    // 1. Derived 析构函数没有被调用！ ❌ ❌
    // 2. 派生类中分配的内存 (data) 没有被释放，造成了【内存泄漏】。
}
```

如果使用虚析构函数，就可以完美的解决上面的问题
1. 运行时机制识别出ptr实际指向Derived对象
2. 首先调用Derived的析构函数，释放派生类资源
3. 然后自动调用Base析构函数
4. 资源被正确释放，没有内存泄露

> [!info]
> 记住黄金法则，什么时候用`virtual`析构函数
> 1. 基类 & 基类中有虚函数， 用`virtual`析构函数
> 2. 基类 & 会被动态函数，也就是用到`new`或者`delete`  那么用`virtual`析构函数
> 3. 如果类是封闭的，没有被继承，可以不使用虚析构函数
> 在实践中，为了安全起见，基类基本都被设置为虚析构函数

## 4.3 补充一下虚函数的底层原理

核心底层原理是 **虚函数表（VTable）** 和 **虚函数指针（VPtr）**


1. 虚函数表
	1. 位置： VTable是一个静态的，由编译器在编译时为每个包含虚函数的类单独创建的函数指针数组
	2. 内容：这个数组的每个元素都是函数指针，指向该类中所有虚函数的实际地址
2. 虚函数指针
	1. 位置： 只要一个类包含或者继承了虚函数，那么它的每个对象中都会在内存布局的起始位置添加一个隐藏指针，这个指针就是虚函数指针
	2. 作用： VPtr指向该类对应的虚函数表的地址

# 5. 纯虚函数

在C++中没有Java的抽象类，只能通过纯虚函数来表达抽象类的逻辑，如果一个类中有纯虚函数，那么这个类就可以认为是抽象类，不能直接实例化，只能作为基类

## 5.1 纯虚函数的用法

纯虚函数是在基类声明一个虚函数，并在末尾使用` = 0` 来标记

```cpp
virtual 返回类型 函数名(参数列表) = 0;
```

```cpp
class Shape {
public:
    // 这是一个纯虚函数，意味着 Shape 类必须是抽象类
    // 派生类必须实现 draw() 方法
    virtual void draw() = 0; 

    // 抽象类中可以有普通的虚函数和非虚函数
    virtual double area() { return 0.0; } 
    void description() { /* ... */ } 
};

// 派生类必须实现 draw()
class Circle : public Shape {
public:
    void draw() override { 
        std::cout << "Drawing a Circle" << std::endl;
    }
};

int main() {
    // 实例化派生类
    Circle c; 
    
    // 通过基类指针实现多态
    Shape* s_ptr = &c;
    s_ptr->draw(); // 输出：Drawing a Circle
    
    return 0;
}
```

需要注意的是，在派生类里面是否需要`override`这个关键字，这个关键字不是必须的，但是这样增强代码的可读性

# 6. 虚基类

## 6.1 什么是虚基类

它主要解决的是多重继承下的『菱形问题』，使用虚基类来解决在多重继承下的**二义性**和**冗余**

看一个场景

1. `Person`是一个基类
2. `Teacher`和`Student`都是`Person`的子类
3. 现在有个类`Assisant` 是`Teacher`和`Student`的子类

如果没有虚基类，会导致一个问题
1. `Assisant`通过`Teacher`得到`Person`的成员
2. `Assisant`通过`Student`得到`Person`的成员

这个就会导致`Assisant`对象有两个`Person`副本，产生了冗余；而且访问person的成员的时候，不知道是从teacher路径还是student路径，编译器直接懵逼了，所以导致了**二义性和冗余**

```cpp
class Person {
public:
    int age;
};

// 关键：使用 virtual public 继承
class Teacher : virtual public Person { /* ... */ }; 

// 关键：使用 virtual public 继承
class Student : virtual public Person { /* ... */ }; 

class Assistant : public Teacher, public Student { /* ... */ };
```

# 7. 模板类（泛型）

## 7.1  模板类基本使用
其实模板类就是实现类的泛型的手段，通过模板类来实现泛型编程。至于什么是泛型，学过Java的就懂了。

```cpp
// 语法
// container.h

template <typename T>
class MyContainer {
public:
    void printValue(); // 成员方法声明
    T data;
    // 构造函数
    MyContainer(T v) : value(v) {}
};
// 注意：模板类的所有定义通常都放在头文件中，但我们这里演示外部定义
```

格式一定要是上面的那种方式，这就是一个通用的类的结构，将参数类型作为参数传递给这个类， 这个T是一个类型参数，可以代表任何类型，int / double/ std::string等

> [!danger]
> 在定义模板参数的时候，你可以使用`typename`或者`class`，它们的含义是一样的，都表示后面的`T`是一个类型参数
> 下面两种写法，完全等价
> `template <typename T> class ClassA {/* .......*/};`
> `template <class T> class ClassA {/*........*/};`
> 实际开发中，更加推荐使用`typename`，这个更加直观和清晰，不会让人误解`T`是一个类

实例话的时候，在使用模板类的时候，必须在类名后面用尖括号`< >`来指定具体的类型

```cpp
// 实例化一个存储 int 类型的容器
MyContainer<int> intContainer(10); 
// 或者这样
MyContainer<int> intContainer = MyContainer<int>(10); 
std::cout << intContainer.getValue() << std::endl; // 输出 10

// 实例化一个存储 double 类型的容器
MyContainer<double> doubleContainer(3.14);
std::cout << doubleContainer.getValue() << std::endl; // 输出 3.14
```


<span class="b">特别补充一个，模板可接收非类型参数（比如整数、枚举、指针等）</span>
```cpp
// SIZE 是一个整数，在编译时确定
template <typename T, size_t SIZE>
class MyArray {
    T data[SIZE];
    // ...
};

// 实例化时指定类型和大小
MyArray<int, 10> arr;
```

## 7.2  模板类的成员方法定义时候的注意事项

在C++规范中，模板类的成员方法，在定义的时候，前面同样必须要加上`template`声明

规范要求，在类外部定义模板成员方法
1. 重复完整的模板参数类表`template <typename T, ....>`
2. 指定该函数属于哪个模板类 `ClassName<T>::`

```cpp
// container.h

template <typename T>
class MyContainer {
public:
    void printValue(); // 成员方法声明
    T data;
};
// 注意：模板类的所有定义通常都放在头文件中，但我们这里演示外部定义

// container.cpp (或头文件中的实现部分)

// 必须重复完整的模板声明
template <typename T>
// 指定是哪个模板类中的方法
void MyContainer<T>::printValue() {
    std::cout << "The value is: " << data << std::endl;
}

// ----------------------------------------------------
// ❌ 错误写法 (不完整)
// void MyContainer::printValue() { 
//     // 编译器不知道 T 是什么，会报错
// }
// ----------------------------------------------------
```

‼️‼️‼️‼️
只有成员方法的定义直接写在类定义内部，才不需要重复的`template`声明。

## 7.3 C++的泛型和Java中的泛型的区别是啥

C++模板和Java泛型在底层设计和哲学上有着根本的区别

| 特性       | C++ 模板                           | Java泛型                                       |
| -------- | -------------------------------- | -------------------------------------------- |
| 底层机制     | 编译时代码生成                          | 运行时类型擦除                                      |
| 实现方式     | 宏展开。编译器为每种类型参数生成一套全新的、独立的字节码/机器码 | 统一类型。运行时所有泛型都退化为Object                       |
| 性能       | 高性能。运行时没有类型转换                    | 运行时需要进行额外的装箱/拆箱和类型转换，<br>性能有损耗               |
| 对基本类型的支持 | 支持原生的基本类型 （`std::vector<int>`）   | 不支持原生基本类型。必须使用包装类，<br>(`ArrayList<Integer>`) |
| 语法复杂性    | 高，必须要在header中暴露所有代码，且处理模板比较复杂    | 低，语法简洁，易于编写和阅读                               |


## 7.4 如何对泛型T进行约束

在Java中，我们发现对T约束很简单

```java
// Java 明确约束 T 必须是 Comparable 接口的实现
public <T extends Comparable<T>> T findMax(List<T> list) { ... }
```

在C++20之前，主要依赖**隐式接口**和**元编程技巧**来实现类似约束

```cpp
#include <type_traits>

template <typename T>
typename std::enable_if<std::is_base_of<Person, T>::value, void>::type
processPerson(T obj) {
    // 只有 T 继承自 Person 时，这个模板函数才可用
    std::cout << "Processing a Person derived object." << std::endl;
}
// ...
```

但是在C++20之后，引入了`Concepts`，彻底改变了模板约束的局面。
`Concepts`允许你定一个命名规则集，然后用和这个规则集来约束模板参数

```cpp
#include <concepts>

// 定义一个 Concept
template <typename T>
// 🔹🔹🔹 用于检查T是不是Person或者其派生类
concept IsDerivedFromPerson = std::is_base_of_v<Person, T>;

// 在模板中使用 Concept 来约束 T
template <IsDerivedFromPerson T>
void process(T obj) {
    // 只有 T 继承自 Person 时，函数才可用
    std::cout << "C++20: Processing a Person derived object." << std::endl;
}

// 定义一个 Concept，要求类型 T 必须支持 == 运算符
template <typename T>
// 🔹🔹🔹 用于检查T 是否支持 == 运算符。这个很强大
concept EqualityComparable = requires(T a, T b) {
    { a == b } -> std::convertible_to<bool>;
};

// 约束 T 必须是 EqualityComparable
template <EqualityComparable T>
bool compare(T a, T b) {
    return a == b;
}
```
