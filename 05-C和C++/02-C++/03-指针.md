
# 1. 和C语言中的指针比较

C++的指针在**核心机制以及内存表示上**和C语言是相同的，因为C++继承了C语言的底层内存模型。
## 1.1  核心机制的相同点
- **内存地址** ： 指针本质上就是存储内存地址的变量，在32位机器上占用固定大小的内存（4个字节），64位上则是8个字节
- **解引用**  都是通过`*`运算符解引用指针以访问它指向的值
- **取地址**   都是通过`&`运算符获取变量的地址
- **指针运算**  相同的逻辑，加减运算都是基于指针类型的大小进行移动
- **void指针**    `void*` 在两种语言中都是用于存储任何数据类型的地址，但是在C++中不能直接解引用，需要类型转换

## 1.2 关键区别

C++对指针的使用有更加严格的限制或者引入了更加现代的替代品，以提高代码的安全性和表达力。

1. 严格的类型检查

C： 允许将`void*`隐式的转换为任何其他数据类型的指针

```c
int* ip = malloc(sizeof(int));
char* cp = ip; // ❌ C语言会警告，但许多编译器可能允许
void* vp = ip;
char* cp2 = vp; // ✅ C语言允许隐式转换
```

C++： 不允许将`void*`隐式转换为其他指针类型。所有指针类型之间的转换都需要显式的类型转换

```cpp
int* ip = new int;
void* vp = ip;
char* cp2 = static_cast<char*>(vp); // ✅ 必须使用显式转换
char* cp3 = vp; // ❌ C++ 编译错误
```

## 1.3 空指针

C正常来说，空指针用`NULL`来表示
但是C++11后，引入了`nullptr`关键字，这个**专用的空指针常量**，具有强类型安全，能够有效的避免`NULL`和整数`0`之间的歧义问题。现代C++开发，强烈建议使用`nullptr`

## 1.4 指针的危险

```cpp
int *p;
*p = 10;
```

这个代码是非常危险的，虽然我们可能侥幸能够正常运行，但是危害很大
- 指针p尚未初始化，它指向一个不确定的、随机的内存地址
- `*p` 没有指向明确的地址，虽然可能`*p= 10`能够成功，但是有很大概率是不成功的，它随机指向的那个地址可能是只读的，怎么办呢？就会出现崩溃异常的。
- <span class="r" style="font-size:20px;background-color:yellow">在指针使用的时候，一定要给指针初始化一个确定的、适当的地址</span>

# 2. new和delete

在C语言中，分配内存采用的是`malloc`这些函数，在C++中提供了一个更加安全的方式`new`

```cpp
int *p = new int;
int *arr = new int[4];

delete p;
delete[] arr;
```

> [!danger]
> 只有通过`new`方式分配的内存，才可以用`delete`。 如果`new`分配的内存是用`new []`的方式，回收内存的时候，也需要用`delete []`。如果用`delete`的话，只会回收第一个元素的空间，从而导致其他空间没有被回收，会导致内存泄露

需要注意的是
```cpp
int arr[] = {23, 4, 5, 6};
// 这个是不合法的，因为arr是常量不能进行加减运算
// arr = arr + 1; ❌ ❌ ❌
```


# 3. 自动存储、静态存储以及动态存储

根据用于分配内存的方法，目前C++有三种管理数据内存的方式： **自动存储、静态存储、动态存储**

## 3.1 自动存储

这个很好理解，我们把一些局部变量就认为是『自动存储』的变量
- **分配位置**    ： 栈  Stack
- **生命周期**    ：自动管理， 在代码块中生效
- **默认初始化**   ： 不保证初始化，如果不显式的初始化，变量包含一个随机的『垃圾』值
- **关键字**         ：无需关键字，  旧标准中可以使用`auto`，但是现代C++的auto的语义发生了变化，主要用于类型推导

## 3.2 静态存储

静态存储是贯穿整个程序执行期间都存在的存储方式。
- **分配位置** ：  数据段 Data Segment  或者BSS段
- **生命周期**  ：程序启动时分配，程序终止时销毁，生命周期贯穿整个程序的运行时间
- **默认初始化**   <span class="b">保证初始化</span>  如果没有显式初始化，会给出一个默认值
- **关键字/位置** 
	- `static` 关键字修饰的局部变量
	- 全局变量，也就是函数外定义的那些
	- 命名空间作用域中的变量

## 3.3 动态存储

动态存储主要用于在运行时『手动』分配和释放的内存

- **分配位置**     : 堆  Heap
- **生命周期**     ：需要手动管理，从`new`开始，到`delete`释放时结束
- **默认初始化**  ：不保证初始化，需要显式的指定
- **关键字**     ： `new`  `delete / delete[]`

## 3.4 线程本地存储期

这个是C++11后新增的存储期，用于多线程

- **分配位置**   每个线程的本地存储区
- **生命周期**   ： 线程启动时分配，线程结束时销毁
- **关键字**  ：  `threal_local`

> [!note]
> 上面在静态存储提到的  『数据段』以及『BSS』 是啥？
> - 它们既不存储在Stack也不在Heap上，是一个独立的区域。
> - 数据段：存储所有已经初始化的全局变量和静态变量
> 	- 只有被显式初始化为非零值的静态变量才会被放入数据段
> - BSS段：存储所有未被初始化或者初始化为0的全局变量和静态变量


# 4. 智能指针

智能指针是C++11之后引入的，用于管理动态分配内存（即使用`new`创建的对象）的一种**类封装指针**。
它是C++实现RAII原则的核心工具。

## 4.1 什么是智能指针

智能指针是<span class="r">行为类似原始指针的类对象</span>，通过重载`*`和`->`运算符，使其可以像普通指针一样使用。
但最大的区别在于：
> 智能指针自动管理所指向对象的生命周期。当智能指针对象被销毁时，它会自动调用`delete`来释放所管理的内存

## 4.2 解决了什么问题

1. 内存泄露
	1. 使用原始指针new一个对象后，如果忘记调用`delete`或者程序出现异常，没有来得及delete，最终会导致内存泄露
	2. 智能指针是一个局部栈对象，无论程序是以正常路径退出作用域，还是通过异常进行栈展开，智能指针的析构函数都会被掉哦那个，从而安全地释放其内部的动态内存
2. 二次释放
	1. 多个原始指针可能同时指向同一块区域，如果其中一个指针调用`delete`，会导致其他指针成为悬挂指针，如果这些悬挂指针再次调用`delete`，会导致程序崩溃
	2. `unique_ptr`确保独有所有权，而`shared_ptr`使用引用计数指针，只有当最后一个`shared_ptr`离开作用域时才会释放内存，完美避免二次释放
3. 悬挂指针
	1. 当内存释放后，原始指针仍然存在，会导致问题
	2. 智能指针将内存挂历和指针生命周期紧密绑定，提高了代码的安全性

## 4.3 C++标准库的三种智能指针和使用场景

```cpp
void unique_owner() {
    // 推荐：安全且简洁地创建 unique_ptr
    // 模板参数是资源类型，构造函数的参数直接传给 MyResource 的构造函数
    auto ptr1 = std::make_unique<MyResource>(101); 

    ptr1->operate(); // 使用 -> 运算符访问成员

    // 所有权转移（Move）
    // ptr1 失去资源，ptr2 获得资源。这是 unique_ptr 唯一允许的转移方式。
    auto ptr2 = std::move(ptr1); 

    if (!ptr1) {
        std::cout << "ptr1 is null after move." << std::endl;
    }
    
} // <--- ptr2 超出作用域，MyResource(101) 被自动 delete。
```
