# 2. 异常

这个跟Java是很像的，都是`try-catch`  以及` throw`抛出异常

## 2.1 异常机制的核心三要素

1. `try`  包含一段可能出现异常的代码块
2. `throw`  当程序要发生错误的时候，主动抛出一个异常对象
3. `catch`  捕获异常，紧跟在`try`块之后，用于捕获异常并进行特定的处理

```cpp
try {
    // 1. 受保护的代码块：可能引发异常的代码
    int result = divide(10, 0); // 假设这里会抛出异常
} 
catch (const std::exception& e) {
    // 2. 异常处理块：捕获标准异常
    std::cerr << "Caught standard exception: " << e.what() << std::endl;
}
catch (int e) {
    // 3. 异常处理块：捕获 int 类型的异常
    std::cerr << "Caught custom error code: " << e << std::endl;
}
catch (...) {
    // 4. 捕获所有异常 (Catch-All)：处理未匹配的异常
    std::cerr << "Caught an unknown exception." << std::endl;
}
```

## 2.2 如何抛出异常

`throw`可以抛出任何类型的对象，但是强烈建议抛出继承自`std::exception`的异常对象

```cpp
#include <stdexcept>

int divide(int numerator, int denominator) {
    if (denominator == 0) {
        // 抛出一个标准库异常 (运行时错误)
        throw std::runtime_error("Attempted to divide by zero!");
    }
    return numerator / denominator;
}
```

> [!danger]
> C++中的异常，可以`throw`任何类型的对象数据，可以是字符串，可以是int等


正常使用的情况下，我们会使用自定义的异常类，需要继承自`std::exception`

```cpp
class MyException : public std::exception
{
public:
	// 重写what函数
    const char *what() const noexcept override
    {
        return "自定义的异常类抛出的异常.....";
    }
};
```

# 3. RAII

在上面，我们发现C++中没有Java的`finally`处理机制。
那么在C++中如何实现`finally`功能呢？
主要是依赖 『Resource Acquisition Is Initialization』 简称 `RAII`

RAII是C++处理资源清理的标准和最佳实践。它利用了C++保证在**栈展开**过程中会调用局部对象的析构函数这一特性

方法： 将需要释放的资源封装在一个类的析构函数中。

```cpp
#include <iostream>
#include <memory>

void process_data() {
    // 使用 RAII 封装的智能指针
    // 无论 try 块是否抛出异常，data 都会在作用域结束时被 delete
    std::unique_ptr<int[]> data(new int[100]); 
    
    // 使用 RAII 封装的锁
    // 无论 try 块是否抛出异常，lock 都会在作用域结束时解锁
    // std::lock_guard<std::mutex> lock(my_mutex); 
    
    try {
        // 核心逻辑...
        if (/* 发生错误 */) {
            throw std::runtime_error("Error during processing.");
        }
        // ...
    } // <-- 无论是以何种方式退出 try 块，RAII 对象的析构函数都会被调用！
    catch (const std::exception& e) {
        std::cerr << "Caught: " << e.what() << std::endl;
        // 异常被捕获，然后继续栈展开
    }
    // 资源在函数退出时自动释放
}
```

# 4. RTTI

这个是 运行时类型信息 Rutime Type Information，主要用于类型的转换，两个操作符`dynamic_cast`和`typeid`

## 4.1 dynamic_cast  安全向下转换

```cpp
class Base { virtual void func() {} };
class Derived : public Base { public: void derivedFunc() {} };

Base* ptr = new Derived;

// 尝试安全地转换为 Derived*
Derived* d_ptr = dynamic_cast<Derived*>(ptr); 

if (d_ptr) {
    d_ptr->derivedFunc(); // 成功转型，可以调用派生类特有方法
}
```

## 4.2 typeid  获取类型信息

```cpp
Base* ptr = new Derived;

// 获取运行时 ptr 指向的实际类型信息
std::cout << typeid(*ptr).name() << std::endl; 
// 输出可能是 "Derived" 或其编码形式
```
