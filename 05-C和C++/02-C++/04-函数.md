
C++的函数和C语言中的函数是一样的，没啥好讲的。

但是在函数的声明上，有稍微的区别

- C++函数，消除了空参数的歧义，也就是`int fun();`   这种就是代表函数体明确表示不接受任何参数。它和`int fun(void);`是等价的
- C语言中，如果旧标准的C++是`int fun()`，它只声明了函数的返回类型，**对参数列表不做任何约束**，只会在运行时进行匹配，在编译阶段无法捕获参数不匹配的问题。
	- 现代C语言中，如果表示函数不接受参数，必须明确表达出来，`int fun(void);`这个是C语言中表达『无参数』的规范方式

# 1. 说明一下const修饰的函数参数

我们就知道在C++中，`const`代表的是常量以及只读，它的核心作用是提高代码的可靠性，清晰度以及效率

被`const`修饰的参数，都是不能修改的
- 用`const` 修饰基本类型的参数，不能修改
- 用`const`修饰`array`，里面的元素都不能修改
	- 因为在C/C++中，arr作为参数，会退化成一个指针，所以`const int arr[]  ==  const int *arr`
	- <span class="r">需要注意的是指针指向的内容是常量：不能修改指针指向的内存元素</span>
	- <span class="r">但是指针本身不是常量，可以修改指针本身，让它指向别错</span>

```cpp
void fun(const int arr[], int size) {
    // 1. 修改数组元素：
    // arr[0] = 10;   // 编译错误！不能修改 const int* 指向的内容❌ ❌ ❌

    // 2. 修改指针本身：
    arr = arr + 1;  // 正确！指针本身不是 const，可以指向下一个元素
    
    // 3. 读取元素：
    int value = arr[0]; // 正确！可以读取元素
}
```

同样的，如果用`const`修饰struct或者class的参数，同样的也是不能修改里面的成员变量，
只有一个例外，如果类中的某个成员变量被`mutable`关键字修饰，那么即使使用`const`修饰的，依然可以修改这个`mutable`成员变量。


> [!danger]
> 那么用`const`修饰的参数和Java中用`final`修饰的参数有啥区别呢？
> `final`的作用：该变量在初始化后不能再被重新赋值，这个跟`const`的行为是一致的
> 但是在`final`修饰对象的时候，他只是保证该引用本身不会改变，但是对象内部的成员变量可以被修改(除非成员变量本身是final的)
> 总结：
> C++ 的`const MyClass& obj;`  保证**对象的内容不可变，禁止修改成员变量**
> Java的`final MyClass obj;`     保证**引用的不可变，可以修改成员变量**

# 2. 说明一下auto的作用

C和C++中都有`auto`关键字，但是它们的作用大相径庭。

## 2.1 auto在C中

在C语言中，`auto`用于声明一个局部变量具有**自动存储期**
但是有一个鸡肋的点就是，在C语言中，任何函数内部声明的局部变量默认说就是**自动存储期**， 也就是默认都是`auto`
所以很少使用`auto`

## 2.2 auto在C++中

在C++11之前，C++中的auto为了兼容C，它们的功能是一样的。但是在C++11之后，`auto`的旧有用法被废弃（但保留以兼容旧代码），它被赋予新的作用
- 类型推导   当使用 $\text{auto}$ 声明变量时，编译器会根据变量的初始化表达式自动推导出该变量的类型。

```cpp
auto x = 10;            // 编译器推导 x 为 int (从 10)
auto d = 3.14;          // 编译器推导 d 为 double (从 3.14)
auto s = std::string("Hello"); 
                        // 编译器推导 s 为 std::string
                        
std::vector<int> nums = {1, 2, 3};

// 迭代器类型通常很长，使用 auto 极大地简化了代码
for (auto it = nums.begin(); it != nums.end(); ++it) {
    // ...
}

// 尤其常用于 lambda 表达式的存储
auto lambda = [](int val) { return val * 2; };
```

# 4. C++内联函数

之前C语言学习的时候，已经学过这个内联函数了，今天再次梳理一下。
内联函数就是为了解决函数的调用的开销，毕竟需要进行入栈和出栈。如果是内联函数的话，在编译阶段就会用对应的函数进行替换，这样就不会出现函数的调用开销了。但是也有代价，会占用更多的内存，毕竟一个函数如果有100处地方使用的话，最终编译的文件就会大一点，毕竟会有100个函数的副本在里面。

> [!danger]
> 1. 使用`inline`函数必须将函数声明和定义都要加上关键词`inline`。
> 2. 正常的做法，不需要声明和定义分开，直接一次性声明和实现体放在一起
> 3. 编译器有权决定是否内联，这个是编译器的一个选择，并不是你想内联他就给你编译成内联了，最终可能就是一个普通的函数

```cpp
inline double square(double x)
{
    return x * x;
}
```


# 5. 引用变量

C++新增了一个引用变量。引用变量其实就是已经定义的变量的别名而已。最常用的场景就是『函数的参数』

```cpp
int a = 2;
int &b = a;  // 这个b就是a的别名
```

一般我们认为引用就是指针的一种简写   `int &b = a;`  就等于  `int *const b = &a`

在函数中使用引用参数，目的是避免昂贵的对象复制。那么，引用参数和指针参数有啥区别呢？

| 特性   | 引用参数(Type &)                   | 指针参数（Type*）                   |
| ---- | ------------------------------ | ----------------------------- |
| 语法   | 直接使用函数名，与操作变量本身没有区别            | 必须使用解引用操作符`*`来访问变量的内容         |
| 空值   | 不允许为空，引用必须是i个有效的对象             | 可以为空，在使用的时候进行非空判断             |
| 重新指向 | **不可重新绑定**，一旦指定初始化，引用始终指向同一个对象 | **可重新赋值**。指针可以指向其他对象或者设置为NULL |
| 安全性  | 更高。避免了空指针的风险和复杂的指针操作           | 较低。需要手动内存管理和空值检查              |
| 可读性  | 更高，代码看起来更加的简洁自然                | 较低，涉及到各种`*`和`->`              |
结论：
- **首选引用**  当我们需要通过参数来改变变量且变量一定存在的情况下，用引用，这样会更加安全，可读性也更加高
- **次选指针**  如果要表达参数可以为空的情况下，用指针。

# 8.  函数模板

现代C++编译器实现了新的功能---**函数模板**。函数模板是通用的函数描述，它不是真正的函数，它使用了泛型来定义函数。在编译的时候，编译器会根据你传入的实际数据类型，自动生成一个该类型特定的函数版本，这个过程为**模板实例化**

## 8.1 语法结构

函数模板使用`template`关键字和类型参数（通常是`typename`或者`class`后跟一个标识符，如`T`）来定义

```cpp
template <typename T>
T max(T a, T b)
{
return (a > b) ? a: b;
}
```

函数模板主要用于编写**类型无关的通用算法**，以实现代码复用。

在容器或者集合的通用操作中，会有大规模的泛型，所以就会有大量的函数模板。同样，函数模板也是支持函数的重载的。
