
# 1. 类的内存分配

其实类的内存分配逻辑和struct是相同的，类的对象在内存分配中，主要是为了存储其==**非静态成员变量**==和一些必要的**开销**（如虚函数表指针）

## 1.1 实例对象的内存分配

| 内存区域    | 示例代码                           | 存储内容                   | 生命周期            |
| ------- | ------------------------------ | ---------------------- | --------------- |
| 栈       | `MyClass obj;`                 | 存储obj的非静态成员及其虚表指针      | 作用域结束时自动释放      |
| 堆       | `MyClass *pt = new MyClass();` | 存储new出来的对象和非静态成员及其虚表指针 | 必须手动`delete`释放  |
| 静态/全局区域 | 存储所有类的静态成员变量                   | 存储所有静态成员               | 程序开始时分配，程序结束后释放 |

## 1.2 对象大小的计算

一个C++的类的对象大小由以下因素决定，这个跟struct很像
1. 非静态成员变量的大小
2. 内存对齐
3. 虚表指针  ： 如果类中包含虚指针，对象内部会有一个隐藏的指针，指向该类的虚函数表，它占用4或8个字节（取决于系统）


# 2. static关键字在类中用法

`static`关键字用于改变成员变量和成员函数的存储位置和行为，使其**不依赖于任何特定的对象实例**


1. 静态成员变量存储在 **静态全局数据区**，与对象实例分离
2. 所有的对象实例共享同一个静态成员变量的副本
3. 在程序启动时分配，只分配一次
4. 访问方式，直接通过类名访问`ClassName::staticVar`
5. 必须要在类定义外部进行定义和初始化，声明可以放在类的定义中

```cpp
class Counter {
public:
    static int count; // 声明：静态成员变量
    Counter() { count++; }
};
```

```cpp
// 定义和初始化：只执行一次
int Counter::count = 0; 

// 在 main 或其他地方使用
// Counter c1, c2; 此时 Counter::count == 2
```


**静态成员函数**

访问限制，不能范围非静态成员变量和调用非静态成员函数

# 2. 特殊的成员函数

C++会自动提供一些成员函数
1. 默认构造函数 
2. 默认析构函数
3. 复制构造函数
4. 赋值运算符
5. 地址运算符


和Java一样，如果提供了一个构造函数，默认的非空构造函数就不会自动生成。

## 2.1 复制构造函数

复制构造函数用于将一个对象复制到新创建的对象中。
假设 `stu`是`Stu`对象，则下面声明将调用复制构造函数

1. `Stu copy(stu);`
2. `Stu stu2 = stu;`
3. `Stu stu2 = String(stu);`
4. `Stu *stu2 = new String(stu);`

在类中可以显式的创建一个复制构造函数，可以处理自己的一些逻辑

```cpp
ClassName(const ClassName &);  // 参数必须是一个引用
```

在复制构造函数中，涉及到一个问题，就是『深拷贝』和『浅拷贝』的问题。
这个问题，在Java中应该就会，此处就不讲了。
