
# 1. 运算符重载

运算符重载是C++中一种实现**多态性**的机制。 就是将原有的运算符重新赋予新的含义和操作逻辑。

核心目的：
- 增强函数的可读性和表达能力

想象一个常见场景，两个对象能够相加吗？ 正常情况下，像个对象相加是啥呢? 如果我们赋予这个操作符`+`的逻辑。

## 1.1 如何实现

在C++中，运算符重载是通过定一个一个新的特殊函数来实现的，这个函数的名是`operator`关键字后接要重载的运算符。

#### 1.1.1 作为成员函数实现

当重载**一元运算符**（比如 `-`、`++`）或者左侧操作数必须是类对象的二元运算符（如`+`、`+=`）时，通常使用成员函数
- 语法： `ReturnType operatorX(Args)`  其中`X`就是操作符
- 左侧操作符： 隐式地作为调用对象（通过this指针）

```cpp
class Vector2D {
public:
    int x, y;
    // 重载 + 运算符：Vector2D operator+(const Vector2D& other);
    Vector2D operator+(const Vector2D& other) const {
        // 创建一个新的 Vector2D 对象作为结果
        return Vector2D(x + other.x, y + other.y); 
    }
};
// 使用：Vector2D V3 = V1 + V2;
// 编译器将其转换为：Vector2D V3 = V1.operator+(V2);
```

针对上面的代码，我有几个问题？ 

1. 1️⃣ ==为啥在函数中会有`const`，尤其函数后有一个`const`==

在C++中，我们有就近原则，看`const`靠近谁，那么就修饰谁

```cpp
// 1. ❗️ ❗️  const放在最前面，就是修饰返回值，表示返回一个不可修改的对象
const Vector2D operator+(const Vector2D& other){} 

// 2. 💯 💯  const放在参数类型之后，代表的是修饰函数本身，表明它是一个常量成员函数
Vector2D operator+(const Vector2D& other) const {} 
```

⚠️⚠️⚠️ 常量成员函数表示 <span class="b">该函数不会修改对象状态，也就是说`this`指向的对象不能修改</span>

```cpp
// 格式： 返回值类型 函数名 (参数列表) const
Vector2D operator+(const Vector2D& other) const 
//                                         ^^^^
// 这里的 const 是修饰整个成员函数，它保证了函数体内不会修改 *this 对象（即左侧的 Vector2D）
```

2. 2️⃣  ==运算符重载函数必须返回类吗？ 参数必须只能一个参数吗？ 是否可以为空或者多个？==

以上的规则，取决于操作符是`一元操作符`还是`二元操作符`。

> [!info]
> 什么是`一元操作符`和`二元操作符`？
> 很简单，就是这个操作符需要几个操作数？  比如`+`，既可能是`一元`，也可能是`二元`，因为`+`可以是正号， `+a` 正数a，`a + b`  此时就是`二元`

运算符重载函数不一定要返回类型，看需要实现的逻辑。

1. **二元运算符**，比如(`+`, `-`,`*`,`+=`等)
	1. 参数数量必须是：有且只有一个显式参数
2. **一元运算符** 比如（`++`等）
	1. 参数数量： 必须为空

有一个特殊的情况，因为我们知道`++`或者`--`  ，这个可以是前置后者后置`a++`或者`--a`;  那么这两种怎么办呢？
C++中规定：==后置的形式==的成员函数必须要接收一个==无用的占位符`int`参数==

```cpp
// 前置 ++A：没有参数
Vector2D& Vector2D::operator++() { /* 逻辑 */ return *this; }

// 后置 A++：有一个占位符 int 参数
Vector2D Vector2D::operator++(int) { 
    Vector2D temp = *this; // 记录旧值
    ++(*this);             // 执行自增
    return temp;           // 返回旧值
}
```

#### 1.1.2  作为非成员函数的实现

如果不想将操作符重载函数放在类中成为成员函数，也可以放在类外。

一般用于操作符左侧的操作数不是我需要的类类型，此时只能使用『非成员函数』的操作符重载

```cpp
class Vector {
    int x, y;
    // ...
    // 作为成员函数，只能处理 Vector + int
    // Vector operator+(int value);
};

// 作为非成员函数实现，可以处理 int + Vector
Vector operator+(int lhs, const Vector& rhs) {
    // int 是左操作数 (lhs)
    return Vector(lhs + rhs.x, rhs.y); 
}

// 调用：Vector v3 = 10 + v1; // 正常工作
```

其实，非成员函数的操作符重载函数，需要撘配下面讲的『友元』来处理。为啥呢？
如果我们上面的类，`Vector`的成员x和y是私有的话，我们直接`rhs.x`能够访问到吗？ 是无法访问的，所以上面的代码不严谨。

> [!danger]
> 需要注意的是，自定义的运算符操作符，参数一定有一个是自定义的类，不允许两个都是基本类型


# 2. 友元

一般情况下，C++的私有成员，在没有public相关的函数没有提供能力的话，在外部是不能访问类中的私有成员的。
那么我们有什么办法在类外访问私有成员呢？
C++提供了一个概念，**友元**

- 友元函数
- 友元类
- 友元成员函数

## 2.1 创建友元

创建友元函数的第一步是将其原型放在类声明中，并在原型声明前加上关键字`friend`

```cpp
class MyClass {
private:
    int privateData;
    
public:
    // 声明友元函数
    friend int getPrivateData(const MyClass& obj);
};

// 在类外定义友元函数（不需要 friend 关键字）
int getPrivateData(const MyClass& obj) {
    return obj.privateData;
}
```

使用场景：
1. 运算符重载，特别是左操作数不是类对象的情况
2. 需要访问多个类私有成员的函数

> [!danger]
> 在类外面定义友元函数，不需要用`MyClass::`这个限定符

## 2.2 友元类

一个类可以将另外一个类声明成为友元，友元类的所有成员函数可以访问它为友元的类的私有和保护成员。

```cpp
class Engine {
private:
    int horsepower;
    
public:
    Engine(int hp) : horsepower(hp) {}
    
    // 声明 Car 为友元类
    friend class Car;
};

class Car {
public:
    void showEngineInfo(Engine& e) {
        // Car 可以访问 Engine 的私有成员
        cout << "马力: " << e.horsepower << endl;
    }
    
    void modifyEngine(Engine& e) {
        e.horsepower = 300;  // 可以修改私有成员
    }
};
```

使用场景：  那些紧密耦合的类，当两个类需要紧密协作的时候

> [!danger]
> 注意事项
> 1. 友元关系不具有传递性：A是B的友元，B是C的友元，但A不是C的友元
> 2. 友元关系不能继承
> 3. 友元关系是单向的：A是B的友元，B不一定是A的友元

## 2.3 友元成员函数

将一个类的特定成员函数声明为另一个类的友元，而不是整个类

```cpp
class Data;  // 前向声明

class Display {
public:
    void show(const Data& d);  // 只声明
};

class Data {
private:
    int value;
public:
    Data(int v) : value(v) {}
    friend void Display::show(const Data& d);  // 友元声明
};

// 在 Data 定义之后才能定义 Display::show
void Display::show(const Data& d) {
    cout << "值: " << d.value << endl;
}
```
