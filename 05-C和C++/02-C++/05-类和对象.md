
从Java过来的， 对类不会感到陌生，而且Java的类也是从C++脱胎而来，学起来应该很快，也就一些不同的名词或者专业术语不一样而已

# 1.  类写在哪儿

既然是类，那么肯定是为了给别人使用的，所以一般不能写在`cpp`文件中，除非这个类只有当前的`cpp`使用，才会出现类在`cpp`中声明和定义。
所以，正常的代码规范，类一定要写在头文件中，这样别人通过`#include`才可以使用
那么，类的声明和定义都要写到头文件中吗？ 不一定，正常应该遵循声明和定义分开的逻辑，除非类的功能很轻量可以这么干

> [!note]
> 如果你要定义的类中只有一些成员变量，建议还是直接使用`struct`吧

总结：
- 类的声明写在头文件中
- 类的方法等定义，写在`cpp`文件中

# 2. 文件名是否要跟类名保持一致？

原则上是可以不保持一致的，但是为了代码规范，文件名尽量和类名保持一致。

```cpp title=Stock.h
#pragma once

#include <string>

class Stock
{
private:
    std::string company;
    double price;
    double total_val;
	void reset_total() { total_val = 1000; }
public:
    void sayHello();
    void buy(long num, double price);
    void update(double price);
};
```

以上就是类的声明，其实就是给出一个轮廓。

> [!note]
> 上面的`private`和`public` 跟Java不太一样，Java是每个成员属性都需要加上，在C++中通过这种类似`Block`的方式，会减少代码的冗余，而且有个类似分层的方式。

# 3. 类的实现

在另外一个单独的`cpp`文件中，直接对声明的类进行相关的实现

```cpp
#include <iostream>
#include "Stock.h"
using namespace std;

void Stock::sayHello()
{
    cout << "Hello, Welcome " << this->company << "\n";
    cout << "Price : " << this->price << "\n";
    cout << "total_val : " << this->total_val << "\n";
    cout << endl;
}

void Stock::buy(long num, double price)
{
    this->price = price;
    this->total_val = num * price;
}

void Stock::update(double price)
{
    this->price = price;
}
```

# 5. 构造函数

跟Java中是类似的

```cpp
// ....此处省略class中的一些信息
public:
	Stock(std::string &_company, double _price, double _total_val);
```

- 构造函数没有返回值，同时也不是`void`

针对构造函数，C++中有很多的细节

## 5.1 参数能否和成员变量一样

是可以一样的，但是如果参数和成员变量一样的话，有很多限制。

1. 在函数体中，不能直接`成员变量 = 参数`; 因为会导致命名遮盖，最后的数据会很诡异
```cpp
Stock(double price) 
{
	price = price;   // ❌ ❌ ❌
}
```

解决办法：
1. 使用`this`指针，在函数体内赋值

```cpp
Stock::Stock(const std::string &company, double price, double total_val)
{
    this->company = company;
    this->price = price;
    this->total_val = total_val;
}
```

2. 参数不同就行，这不是废话吗
3. <span class="b">使用成员初始化列表的方式 ----推荐方式</span>
这是C++中**初始化成员变量的首选和最佳方式**。在初始化列表中，编译器可以自动分区参数和成员变量。

语法： 在构造函数参数列表后，使用冒号`:`引入初始化列表。

```cpp
// 构造函数声明
MyClass(int a, const std::string& s);

// 构造函数定义 (.cpp 文件中)
MyClass::MyClass(int a, const std::string& s)
    : m_a(a),                // 使用参数 a 初始化成员 m_a
      m_s(s)                 // 使用参数 s 初始化成员 m_s
{
	// 一般函数体是空的
    // 函数体通常留空或用于执行校验和复杂的设置逻辑
    if (m_a < 0) {
        throw std::invalid_argument("A must be non-negative.");
    }
}
```

为啥是最佳实践呢？
1. **效率**： 当你使用初始化列表时，成员变量是直接初始化的。如果在函数体内使用赋值语句，对于非基本类型（比如std::string），会先调用默认构造函数进行初始化，然后再进行赋值操作，效率地下。
2. **强制性**： `const`成员、引用成员和没有默认构造函数的类成员必须通过初始化列表来初始化
3. 对于`const`引用传递复杂对象
	1. 复杂对象，应该使用`const`常量引用。

## 5.2 关于一个构造的问题

```cpp
Stock::Stock(std::string &_company);

Stock stock = Stock("jiangsu");   //  ❌ ❌ ❌
```

使用上面的构造函数，如果用下面的实例化的话，是有问题的，为啥呢？
构造函数的参数是 **非const左值引用**

实参的类型是C风格的字符串字面量，在C++中，它会被用来创建一个临时的`std::string`对象，这个临时对象是一个『右值』。
==所以，构造函数需要左值 ，实参是右值，最终就不匹配==

为啥C++禁止这种绑定？

编译器阻止这种绑定的目的是为了 **安全**

解决办法：
1. 使用`const` 引用

```cpp
Stock::Stock(const std::string &_company);
```

2. 直接使用传值的方式，不用引用就ok，这个就不谈了
3. 在C++又提供了一个方式， 叫『右值引用』
	1. 构造函数内部货缺货这个临时对象的所有权，然后使用

```cpp
Stock::Stock(std::string &&_company);
```

## 5.3 补充一个左值和右值的概念

左值和右值是C++中的核心概念，它们定义了一个表达式的**值类别**。

最简单和经典的区分方式是看**表达式是否有固定的内存地址（即可取址）和 是否具有持久化**

### 5.3.1 左值

左值本质上是一个有持久身份或者可以被取地址的表达式，它们通常是具有名称的变量。
特点：
- **持久身份**    ： 表达式结束后仍然存在
- **可取地址**  ：   可以对它使用取地址操作符 `&`
- **可位于左侧**  ： 传统上，可以作为` = ` 左侧的操作数

```cpp
int a = 5;
int& ref = a; // a 是一个左值，可以被引用绑定
&a;           // 可以取地址
```

### 5.3.2 右值

右值就是没有持久身份或者不能被取址的表达式。通常是临时对象或者字面量。
特点：
- **临时性**   ：  在表达式求值后会被销毁
- **不可取址**   ： 不能对齐使用取地址操作符（临时对象除外）
- **可位于右侧**  ： 只能作为` = `的右侧操作数

```cpp
int b = 10;
b = a + 5; // (a + 5) 是一个右值
// int& ref = 10; // 错误：不能将左值引用绑定到右值 10
```

# 6. 析构函数

析构函数和构造函数类似，区别是
- 没有参数
- 不能重载
- 函数名前有`~`

```cpp
Stock::~Stock()
{
// 相关逻辑
}
```

什么时候调用析构函数呢？这个是由编译器别顶的，通常我们不应该显式调用，虽然显式调用合法，但是危害很大。

- 如果是『静态存储区』的对象，析构函数在程序结束时调用
- 如果是『自动存储区』对象，其析构函数在程序执行完代码块后自动调用
- 如果是new创建的『动态存储区』，在执行`delete`后自动调用

如果没有提供析构函数，就是有一个隐式的默认析构函数。

>[!danger]
>正常情况下，我们**不应该**自己显式的调用析构函数，因为这个由编译器或者运行时环境自动管理。
>如果显式调用的话，可能的后果就是『二次销毁』。
>
>唯一可以主动调用析构函数的场景是 `Placement New`
>`Placement New`允许你将对象构造到一块已经分配好的原始内存上，在这种情况下，你需要手动配对操作
>1. 分配内存     使用`malloc`或者`new`的方式分配原始内存
>2. 构造对象，使用`Placement New`  在这块内存上构造对象
>3. 销毁对象   必须手动显式调用析构函数来销毁对象，因为编译器不知道这块内存上有一个对象
>4. 释放内存   手动调用`free`或者`delete`来释放内存
