## 🌐 一、 HTML / CSS 基础常识（6道）

> 💡 **核心考核点：** 能把页面搭起来、能调明白最基本的样式、知道怎么解决经典布局问题。

1. **什么是 HTML5 的语义化标签？有什么好处？**
* *答：* 就是用有明确含义的标签（如 `<header>`, `<article>`, `<footer>`）代替一堆 `<div>`。好处是代码结构清晰、对 SEO（搜索引擎优化）友好。


2. **CSS 中 `position` 属性常用的几个值（Static、Relative、Absolute、Fixed）有什么区别？**
* *答：* `static` 是默认流；`relative` 相对自己原本的位置定位；`absolute` 相对最近的非 static 祖先元素定位；`fixed` 相对浏览器窗口定位。


3. **在 CSS 中，实现一个元素水平垂直居中，你最常用什么方法？**
* *答：* 现代布局最常用 Flexbox：给父元素设置 `display: flex; justify-content: center; align-items: center;`。
	1. 传统的块级block元素，可以使用 `margin: 0 auto`
		1. 需要注意的是，必须给元素，添加`width`元素，要不然会自动一整行，就无所谓居中了
	2.  要居中行内元素，或者行内块元素，可以使用`text-align: center`
		1. 需要注意的是这个属性要写在父元素中，不能放在自身元素上
	3. 还可以使用经典的绝对定位+位移大法

```css
.parent {
    position: relative; /* 父级作为定位基准 */
}

.child {
    position: absolute;
    left: 50%;          /* 1. 先让子盒子的左边缘对齐父盒子的中线 */
    transform: translateX(-50%); /* 2. 再让自己向左往回位移自身宽度的 50% */
}
```


2. **CSS 盒模型（Box Model）中的标准盒模型和 IE 盒模型（怪异盒模型）有什么区别？怎么切换？**
* *答：* 标准盒模型的 `width` 只是内容（content）宽度；IE 盒模型的 `width` 包含了 padding 和 border。通过 `box-sizing: border-box;` 可以切换成 IE 盒模型（这在现代开发中是标配）。


5. **什么是 CSS 预处理器（如 Sass / Less）？它们解决了什么问题？**
* *答：* 它们让 CSS 可以像写代码一样支持变量、嵌套、函数和模块化，解决了原生 CSS 难以维护、重复代码多的问题。


6. **响应式设计（Responsive Design）的基本原理是什么？**
* *答：* 核心是使用 CSS 的媒体查询（`@media`），根据浏览器窗口的不同宽度，应用不同的样式规则，从而适配手机、平板和 PC。

## 💛 二、 JavaScript / TypeScript 核心基础（12道）

> 💡 **核心考核点：** JS 是前端的核心逻辑语言。作为后端，你需要展现出对异步、事件循环以及现代 TS 类型的理解。

7. **JS 中 `var`、`let` 和 `const` 有什么区别？**
* *答：* `var` 是函数作用域且存在变量提升（不推荐）；`let` 和 `const` 是块级作用域、有暂时性死区；`const` 声明的变量不能重新赋值。

8. **什么是闭包（Closure）？它有什么实际作用？**
* *答：* 闭包就是“一个函数有权访问另一个函数内部变量”的现象。可以用它来创建私有变量、防止全局变量污染。


10. **JS 中的原型（Prototype）和原型链是什么意思？**
* *答：* 每个 JS 对象都有一个原型对象，当访问一个对象的属性时，如果它自己没有，就会去它的原型上去找，一层层往上找的这条链条就是原型链（JS 实现继承的底层机制）。


11. **普通函数和箭头函数（Arrow Function）有什么区别？**
* *答：* 箭头函数语法更简洁，且**没有自己的 `this**`，它的 `this` 继承自外层上下文；另外箭头函数不能用作构造函数（不能 `new`）。


12. **什么是 Promise？它解决了什么问题？**
* *答：* Promise 是处理异步操作的对象，它解决了传统 JS 回调函数（Callback）导致的“回调地狱（Callback Hell）”问题，让异步代码可以用链式调用。


13. **`async/await` 和 Promise 是什么关系？**
* *答：* `async/await` 是 Promise 的语法糖，它能让你用同步的代码书写风格来写异步逻辑，可读性极高。


14. **谈谈你对 JS 事件循环（Event Loop）中微任务（Microtask）和宏任务（Macrotask）的理解。**
* *答：* JS 是单线程的，异步任务分两种。宏任务（如 `setTimeout`、DOM事件）和微任务（如 `Promise.then`）。每一轮事件循环中，**微任务队列会先于下一个宏任务清空**。


15. **什么是深拷贝（Deep Copy）和浅拷贝（Shallow Copy）？怎么实现深拷贝？**
* *答：* 浅拷贝只复制对象的第一层引用；深拷贝会递归复制所有层级。最简单的深拷贝可以用 `JSON.parse(JSON.stringify(obj))`，或者现代浏览器自带的 `structuredClone(obj)`。


16. **TypeScript 中的 `interface`（接口）和 `type`（类型别名）有什么区别？**
* *答：* 它们很相似，但 `interface` 专门用于定义对象结构，支持通过重复声明来自动合并（Declaration Merging）；`type` 更灵活，可以定义联合类型、元组等。


17. **TypeScript 中的 `any` 和 `unknown` 有什么区别？**
* *答：* `any` 会直接绕过所有类型检查（不安全）；`unknown` 也是顶级类型，但你在对它进行操作前，**必须先进行类型断言或类型守卫检查**（更安全）。


18. **什么是 TS 的泛型（Generics）？**
* *答：* 就像 Java 的泛型一样，允许你在定义函数、接口或类时不预先指定具体类型，而是在使用时再动态传入类型，增强代码复用性。

## ⚛️ 三、 React 开发常识（12道）

> 💡 **核心考核点：** React 的核心是“数据驱动视图”和组件化。需要明白状态管理、生命周期的 Hook 替代方案。

19. **React 的虚拟 DOM（Virtual DOM）是什么原理？**
* *答：* 它用一个轻量级的 JS 对象来模拟真实的 DOM 树。当数据变化时，React 先在内存里对比新旧虚拟 DOM（Diff 算法），最后只把发生变化的部分更新到真实 DOM 上，提高性能。


20. **React 中组件的 State 和 Props 有什么区别？**
* *答：* `State` 是组件内部自己管理的可变数据；`Props` 是父组件传递给子组件的只读数据，子组件不能直接修改它。


21. **为什么在 React 中不能直接修改 State（如 `this.state.count = 1`），而是要用 `setState` 或 `setCount`？**
* *答：* 因为直接修改变量无法触发 React 的底层响应式机制，视图不会重新渲染。必须调用修改函数，React 才能监听到变化并更新视图。


22. **React Hooks 解决了什么问题？常用的有哪些？**
* *答：* 它们让函数组件也能拥有状态和生命周期，解决了以前类组件（Class Component）逻辑难以复用、代码臃肿的问题。最常用的是 `useState` 和 `useEffect`。


23. **`useEffect` 的第二个参数（依赖项数组）有什么作用？**
* *答：* 如果不传，每次渲染都会执行；如果传空数组 `[]`，只在组件挂载和卸载时执行一次（类似 `componentDidMount`）；如果传了变量 `[a, b]`，只有当这些变量发生变化时才会重新运行。


24. **在 `useEffect` 中返回一个函数（Cleanup function）有什么用？**
* *答：* 用于在组件卸载或下一次副作用执行前做**清理工作**，比如清除定时器（`clearTimeout`）或取消事件监听。


25. **React 组件中，列表渲染为什么要加一个唯一的 `key` 属性？**
* *答：* 帮助 React 的 Diff 算法快速准确地识别哪些元素被修改、添加或删除了，从而优化 DOM 的复用和渲染性能。


26. **什么是 React 的 Context API？它解决了什么问题？**
* *答：* 它提供了一种在组件树中全局跨层级传递数据的方法，解决了多层组件嵌套时，不得不通过 Props 逐层透传（Props Drilling）的麻烦。


27. **React 中的 `useMemo` 和 `useCallback` 有什么区别？**
* *答：* `useMemo` 用于缓存**计算结果**（避免重复高耗时计算）；`useCallback` 用于缓存**函数定义本身**（避免子组件因为父组件重新渲染而导致函数引用改变，引发不必要的重绘）。


28. **什么是受控组件（Controlled Component）和非受控组件？**
* *答：* 表单元素的值如果完全由 React 的 `state` 控制和驱动，就是受控组件；如果直接通过 `ref` 去读写 DOM 节点的值，就是非受控组件。


29. **React 中父组件如何向子组件通信？子组件又如何向父组件通信？**
* *答：* 父传子通过 `Props` 直接传递；子传父则是父组件传一个**回调函数**给子组件，子组件调用该函数并传入参数。


30. **什么是 React 严格模式（Strict Mode）？为什么我的 `useEffect` 在开发环境下会执行两次？**
* *答：* 严格模式是 React 用来检查组件是否存在副作用的工具。在开发环境下，它会故意连续挂载和卸载组件两次，帮你提前暴露和排查没有正确清理的定时器或监听器。

## 💚 四、 Vue 开发常识（12道）

> 💡 **核心考核点：** Vue 强调响应式和双向绑定。重点是 Vue2 和 Vue3 的 Composition API 区别。

31. **Vue 的双向数据绑定（MVVM）原理是什么？**
* *答：* 视图（View）变化会自动更新数据（Model），数据变化也会自动驱动视图更新。Vue 内部通过数据劫持和发布订阅模式来实现。


32. **Vue 2 和 Vue 3 的响应式原理有什么本质区别？（后端面试官极爱问）**
* *答：* Vue 2 使用 `Object.defineProperty`，无法监听对象新增属性和数组索引变化；Vue 3 改用 ES6 的 `Proxy`，能完美监听任何对象的任何变化，且性能更好。


33. **Vue 3 的 Composition API（组合式API）和 Vue 2 的 Options API 有什么区别？**
* *答：* Options API 把代码按 `data/methods/computed` 拆分，逻辑散乱；Composition API（如 `setup`）允许将相同功能的业务逻辑、状态、函数组合写在一起，更利于代码复用和维护。


34. **Vue 中的 `v-if` 和 `v-show` 有什么区别？**
* *答：* `v-if` 是动态控制 DOM 元素的**创建和销毁**（有更高的切换开销）；`v-show` 只是控制 CSS 的 `display: none;` 隐藏（有更高的初始渲染开销）。


35. **Vue 中的 `computed`（计算属性）和 `watch`（侦听器）有什么区别？**
* *答：* `computed` 具有**缓存特性**，只有它依赖的数据变了才会重新计算，用于简化模板内的表达式；`watch` 用于监听某个数据变化并执行**副作用异步操作**。


36. **Vue 3 中 `ref` 和 `reactive` 定义状态有什么区别？**
* *答：* `ref` 可以定义基本数据类型和对象，在 JS 中访问需要加 `.value`；`reactive` 只能定义对象或数组，不需要加 `.value`。


37. **Vue 中 `v-for` 和 `v-if` 为什么不建议写在同一个标签上？**
* *答：* 在 Vue 2 中 `v-for` 优先级高，导致每次循环都会走一遍 `v-if`，极度浪费性能；Vue 3 中 `v-if` 优先级变高了，容易因为找不到遍历变量而报错。正确的做法是用 `<template>` 标签包裹外层。


38. **Vue 的生命周期钩子中，`mounted` 和 `created` 有什么区别？**
* *答：* `created` 时组件实例已创建，可以获取数据，但**DOM 还没有挂载**；`mounted` 时 **DOM 已经渲染完成**，可以进行 DOM 操作。


39. **什么是 Vue 的 `nextTick`？它的应用场景是什么？**
* *答：* Vue 更新 DOM 是异步批量的。当你修改了数据，想立刻获取更新后的真实 DOM 节点状态时，必须把代码写在 `nextTick` 的回调函数里。


40. **Vue 组件之间跨层级（非父子）通信有哪些常用方法？**
* *答：* 常用方法包括：事件总线（Event Bus）、Provide / Inject 祖孙注入、以及使用状态管理库（Pinia / Vuex）。


41. **Vue 3 中的 `defineProps` 和 `defineEmits` 是做什么用的？**
* *答：* 它们是 Vue 3 `<script setup>` 语法糖里的编译器宏，专门用来在子组件中声明接收父组件的 `props` 和定义要派发给父组件的 `emits` 事件。


42. **什么是 Vue Router 的路由守卫（Route Guards）？常见用途是什么？**
* *答：* 它是路由跳转过程中的钩子函数（如 `beforeEach`）。最常见的用途是**权限拦截**：检查用户是否登录，如果没有登录则强制重定向到登录页。

## 🛠️ 五、 Vite 与前端工程化（8道）

> 💡 **核心考核点：** 能跟后端构建工具（如 Maven, Gradle）产生共鸣的领域。面试官想看你懂不懂前端是怎么打包、怎么联调的。

43. **Vite 为什么比传统的 Webpack 快这么多？**
* *答：* 在开发阶段，Vite 利用了浏览器原生的 **ES Modules (ESM)** 特性，不需要像 Webpack 那样先打包全量代码，而是“按需加载”；此外，它用 Go 语言写的 Esbuild 来预构建依赖，速度是 JS 构筑工具的几十倍。


44. **在 Vite 项目中，你是怎么解决前端跨域（CORS）问题的？**
* *答：* 在 `vite.config.js` 中配置 `server.proxy`。把前端发往 `/api` 的请求，在本地开发服务器端代理转发到后端的真实接口地址（如 `http://localhost:8080`）。


45. **什么是前端环境配置文件（`.env`）？Vite 中怎么读取它？**
* *答：* 它是用来区分开发环境、测试环境和生产环境配置（如后端 API 基地址）的文件。Vite 中要求变量以 `VITE_` 开头，在代码里通过 `import.meta.env.VITE_XXX` 读取。


46. **前端打包产物中的“Tree Shaking”（摇树优化）是什么意思？**
* *答：* 它基于 ES Modules 的静态分析，在打包时**自动剔除那些引入了但从未被实际使用的死代码（Dead Code）**，从而减小最终上线的 JS 文件体积。


47. **什么是单页面应用（SPA）？它有什么优缺点？**
* *答：* SPA 整个网站只有一个 HTML 页面，通过 JS 动态路由切换内容而无需刷新整个网页。优点是体验流畅、前后端彻底分离；缺点是首屏加载慢、对传统的 SEO 不够友好。


48. **当你在浏览器输入一个前端 SPA 路由（如 `/user/profile`）刷新页面报 404，这是什么原因？怎么解决？**
* *答：* 原因是浏览器误以为要去后端服务器请求 `/user/profile` 这个真实的物理文件，但服务器上并没有。解决办法是**配置后端的 Nginx**，将所有找不到的路径全部重定向路由到 `index.html`，把控制权交回给前端路由。


49. **什么是 ESLint 和 Prettier？它们在团队协作中扮演什么角色？**
* *答：* `ESLint` 负责**代码代码质量检查**（查错，比如未使用的变量）；`Prettier` 负责**代码格式化**（风格统一，比如空格、单双引号）。它们能保证团队代码风格绝对一致。


50. **前端性能优化有哪些最基础、稳妥的方法？（说出 3-4 点即可）**
* *答：*
1. 资源压缩合并（代码混淆、图片压缩）。
2. 路由懒加载（异步组件，按需加载）。
3. 合理利用浏览器缓存（强缓存和协商缓存）。
4. 生产环境使用 CDN 托管公共第三方静态资源。
