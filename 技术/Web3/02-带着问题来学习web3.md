
# 1. 什么是比特币？

比特币是 2008年中本聪 提出的**点对点 Peer-to-Peer  P2P**  的点子现金系统

特点： 
- $\text{去中心化}$   它不依赖政府、银行或者金融机构来发行和管理，而是由一个全球性的计算机网络共同维护
- $\text{数字稀缺性}$   比特币的总量固定在**2100W**枚，模仿了贵金属的稀缺性

## 1.1 比特币的底层核心逻辑

比特币的底层核心逻辑是基于 **区块链技术**  + **经济激励模型**

可以认为，比特币就是一个大型的分布式记账系统。

**A.区块链**
1️⃣ ==所有的账本都是公开的==，在整个比特币网络维护着一个公开的、共享的、且持续增长的**分布式账本**，这个就是区块链
2️⃣ ==链式结构==， 交易数据被打包成一个数据块（Block）。每个新的区块都包含前一个区块的Hash值，从而将他们按照时间顺序链接起来，形成一个不可篡改的链。
3️⃣  ==不可篡改性==

**B.经济激励**
1️⃣ ==挖矿奖励==  比特币通过奖励机制来激励参与者维护网络，要不然谁愿意干活呢。成功打包一个新的区块的矿工会获得新发行的比特币以及交易手续费。 这个就是我们说的『挖矿』
2️⃣ ==减半==   区块链奖励大约没4年减半一次。保证了最终供应链的稀缺性

**C.核心机制：UTXO模型**
1️⃣  ==比特币使用UTXO==(Upsent Transaction Output  未花费的交易输出)模型来追踪所有权和余额，这是一种比传统账户模型更加安全、更加简洁的方式

## 1.2 比特币如何保证安全性

比特币的安全性和有效性主要靠三大支柱保证：  <span class="b" style="font-weight:bold">密码学、共享机制以及经济奖励</span>

**A.密码学保证**

- ==数字签名==  每笔交易都需要发送方的**私钥**进行数字签名，保证了只有资产真正的所有者才能够转移资产吗，防止伪造交易
- ==哈希函数==   采用SHA-256这种非对称加密方式来进行hash值运算和参与挖矿的竞争，保证数据的完整性和不可逆性

**B.共识机制： 工作量证明 Pow  Proof of work**

- ==挖矿竞争==    矿工们通过大量的计算来竞争解决一个复杂的数学问题，这个就是工作量证明
- ==验证和安全性==   只有第一个找到正确答案的矿工才能将区块添加到链上。 PoW机制使得攻击者想要恶意篡改历史记录或者双重支付，就必须投入超过全球网络的51%算力，成本极高，几乎不可行，如果真有这个人，他就已经统治地球了，还在乎这个区块链吗？

**C.去中心化和经济奖励**

- ==全球分布式节点==   全球N个节点同时维护着相同的账本副本，如果某个节点试图作弊，其他诚实节点会立即拒绝其无效区块
- ==激励对齐==   矿工的经济利益与维护网络的诚实运行紧密挂钩，只有遵循规则，矿工才能获得比特币奖励

## 1.3 比特币的几个重要的问题

**A.凭啥给你记账和打包block**

进行挖矿奖励，给出对应的奖励和手续费，肯定会有人给你进行记账和挖矿的

**B.如果同时有人打包成功了，应该怎么办**

这个在比特币中的正常现象

1. <span class ="b">初识阶段：临时分叉</span>

现象：两条链同时存在

有两个矿工A和B同时在同一个块后面算出了相同的解，各自打包了一个区块广播了出去（Block-A 和 Block-B）
由于网络差异，不同的节点收到的链，有些节点会先收到Block-A并将其添加到自己的链上；而另一些节点会先收到Block-B并添加到自己的链上

导致的结果就是：此时，比特币网络上暂时分裂成两条链，它们都声称自己是主链，这两条链在N处出现了分叉fork

![[Pasted image 20251123164641.png|600]]
2. <span class ="b">解决机制： 最长链原则</span>

比特币协议中采用一个核心的规则来解决上面的分叉，就是『最长链原则』

在分叉的时候，后续的矿工继续工作，会有两派人分别去构建A链和B链。它们会继续竞争，看谁能够找到下一个区块N+1
假如：链A的矿工更快的找到了N+1块并广播了出去；此时之前正在挖链B的矿工发现链A现在比B更加长，按照协议规则，这些矿工会立即放弃他们正在构建的链B，放弃Block-B，切换到链A上继续工作


3. <span class ="b">最终结果：被抛弃的区块</span>

在fork竞争中失败的那个区块会被网络视为一个孤块，或者过时的区块。
孤块中的交易信息会被重新放回交易内存池中，等待下一个有效的区块进行重新打包


**C.如何防止双花（双重支付）**

比特币中有人会进行双重支付，比如只有10个BTC，但是它同时给A和B都转账了10个BTC，比特币网络是如何解决这个问题的呢？
核心的机制： 交易的唯一性和顺序

1. 因为这个人的账本是共享的，所有人都能够看你的账号的余额以及历史交易记录。
2. UTXO模型检查，来追踪所有权和唯一性检查
3. 以及最终的工作量证明 和 最长链原则，组合起来使用就解决双花的问题


**D.如何计算多久打一次block**

中本聪设置的是每10分钟打包一次，每次奖励50个比特币，同时每过4年奖励的金额减半

```
50 * 6 * 24 * 365 * 4 * (1 + 1/2 + (1/2)^2 + ......)  约等于  2100W个比特币
```

比特币的一个Block大小为1M，而每笔交易差不多250字节甚至更多。

> [!note]
> 1M = 1024kb = 1024 * 1024 个字节
> 由于每笔交易在250字节以上，所以每个block包括 1024 * 1024 /250 = 4194.3笔 记录
> 现在是每10分钟打包一次，所以每秒处理而TPS  就是 4194.3 / 60 = 7   
> 
> ==所以，我们看到比特币的TPS太低了，只有7，这完全没法满足大规模应用==

# 2. 什么是智能合约？

**一句话总结：智能合约 = 部署在区块链上的自动执行的代码**

智能合约像一个机器人，当满足条件时自动执行，无需人工干预，无法篡改。

简单的对比

| 传统系统 <img style="width:250px"/> | 智能合约 <img style="width:250px"/> |
| ------------------------------- | ------------------------------- |
| 代码部署在服务器                        | 代码部署在区块链                        |
| 后端可随时改代码                        | 一旦部署不能修改                        |
| 数据库存储在企业                        | 数据存储在区块链                        |
| 代码由公司空值                         | 代码开源，公开透明                       |
| 费用由公司承担                         | 消耗Gas费用                         |

# 4. 什么是Gas？

在区块链，尤其是ETH EVM链里面： **Gas = 区块链上执行代码的『计算费用』**

在链上运行代码、修改数据、调用智能合约，都需要花钱，这个钱就是Gas
区块链是分布式的，每次执行代码，全球所有节点都必须
1. 同时执行这段逻辑
2. 达成共识
3. 将结果保存进区块
以上的都需要钱。。。。。

为什么可以发行虚拟币？ 因为大多数链允许部署自己的智能合约。

## 6.1 发行虚拟币需要什么？

1. 一个钱包（MetaMask）
	1. 保存你的私钥，支付部署费用Gas
2. 一点点手续费
	1. 在ETH部署一个ERC20，大概需要20-80个 USD Gas
3. 一份基于ERC20合约
	1. 自己写

## 6.2 怎么发布一个虚拟币呢？

目前最主流的方式是 用`openzeppelin` + `Remix`

1. 安装一个钱包
	1. 最常用：MetaMask
	2. 这个是发布合约用到的钱包
2. 准备一点ETH  部署费用
3. 写智能合约
	1. 设置名称、符号、初识发行量
4. 上传代码到Remix网站
5. 选择链并部署合约
	1. 确认MetaMask交易 -> 链上生成一个新的Token合约
6. 合约地址生成，代币已经发布成功
	1. 每个人可以查询代币的余额、转账、添加到MetaMask钱包

# 7. 当前区块链的弊端和不足的地方

区块链的难点从来不是场景，而是技术问题，有多个根本性的技术问题没有得到解决

1. **性能问题，TPS太低，无法大规模应用**
因为区块链必须全网节点同步数据和验证交易，这个就导致TPS很低，延迟很高，吞吐量极低
Visa TPS在2W+，微信支付宝在1W+，以太坊只有15-30，比特币更低，只有7

2. **拓展性差**
无法同时兼顾  去中心化 + 安全 + 高性能

3. **成本高**

链上每次计算都需要费用，普通用户根本用不起

4. **链下的数据不安全，导致无法构建大型应用**
5. **安全风险极高**
智能合约写错代码，由于无法回滚，可能导致巨额亏损
