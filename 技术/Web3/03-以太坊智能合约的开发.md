
# 1. 准备工作

我们所有的开发不可能直接在以太坊的主网络进行测试，因为每一次的交易都是需要花ETH的，所以，我们应该是用以太坊的测试网络进行测试

## 1.1 测试网络
目前主流和推荐的以太坊的测试网络为

| 测试网络名称      | 类型  | 主要用途                                  | 状态            |
| ----------- | --- | ------------------------------------- | ------------- |
| **sepolia** | PoS | **首选网络**。用于DApp开发测试应用、部署合约和进行一般性功能的测试 | ✅ 活跃，但是可能拥堵   |
| **Holesky** | PoS | **大型网络**  主要用于测试以太坊基础设施。 规模更加接近主网     | ❗️已经退役了，尽量不用了 |
| **Hoodi**   | PoS | 最新出的ethereum的测试网络                     | ✅ 活跃          |
|             |     |                                       |               |
|             |     |                                       |               |

## 1.2 配置测试网络和赚钱测试ETH

使用`MetaMask`钱包进行测试，由于上面的测试网络都是基于以太坊的，所以他们的地址和网络上的地址是相同的，因为用的是同一套私钥生成的字符串，所以是相同的，但是在不同网络之间互不影响。

<span style="font-weight:bold; font-size:1.2rem">A.配置Sepolia</span>

直接在MetaMask钱包中可以选择测试网络中，选择 『sepolia』即可。

   **如何赚取Test ETH呢？**
1. 切换到Sepolia网络，然后复制对应的Ethereum的地址
2. 在Google上搜索`sepolia faucet`  , 可以选择基础供应商（如Alchemy或者Infura），但是我选择的是[https://sepolia-faucet.pk910.de/](https://sepolia-faucet.pk910.de/)
3. 把地址粘贴进去，然后等着挖矿就行。

  **查看Sepolia Ethereum的区块信息**
  [https://sepolia.etherscan.io/](https://sepolia.etherscan.io/)

<span style="font-weight:bold; font-size:1.2rem">B.配置Holesky</span>

Holesky需要自己手动在MetaMask钱包中添加网络才可以使用

1. 设置网络名称 ，自己起一个名字，最好直接就是Holesky
2. 需要添加RPC URL地址     从这个网站中找一个[https://chainlist.org/chain/17000](https://chainlist.org/chain/17000
3. Chain ID    固定为`17000`， 代表是HoleSky专用的网络识别码
4. 货币符号   ETH
5. 区块浏览器      [https://holesky.etherscan.io](https://holesky.etherscan.io)  用于查询交易和地址的信息

目前没有找到一个很好的赚取Holesky测试ETH的网站，很多都有限制。

<span style="font-weight:bold; font-size:1.2rem">C.配置Hoodi</span>

Hoodi同样需要在MetaMask上进行手动添加
 1. RPC RUL地址 从网上找一个[https://ethereum-hoodi-rpc.publicnode.com/](https://ethereum-hoodi-rpc.publicnode.com/)
 2. Chain ID  `560048`

可以采用`Sepolia`的那个免费的faucet [https://hoodi-faucet.pk910.de/](https://hoodi-faucet.pk910.de/)

# 2. 开发环境的搭建

现在有很多在线用于学习和验证的IDE，比如Remix。
但是我主要是针对专业程序员的本地的环境的搭建。

## 2.1 安装基础环境

1. 安装Node.js的环境   
2. 安装node的包管理工具
3. 安装vscode的插件
	1. `Solidity`  

## 2.2 安装钱包

使用MetaMask

## 2.3 智能合约框架

目前主流的框架是`Hardhat`，另外也可以是[Foundry](https://getfoundry.sh/)

1. 创建项目目录

```bash
mkdir my-solidity-project
cd my-solidity-project
```

2. 初始化npm项目

```bash
npm init -y
```

3. 安装Hardhat

```bash
npm install --save-dev hardhat
```

4. 初始化hardhat配置

```bash
npx hardhat
```

## 2.4 开发自己的智能合约

在`.sol`文件中写`solidity`源代码，这个就是智能合约的内容，具体可以查看[https://www.soliditylang.org/](https://www.soliditylang.org/)

```sol
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract MyToken {
    string public name = "DemoToken";
    string public symbol = "DTK";
    uint public totalSupply = 0;

    mapping(address => uint) public balanceOf;

    function mint(uint amount) public {
        balanceOf[msg.sender] += amount;
        totalSupply += amount;
    }
}
```

## 2.5 编译和测试

1. 编译

```bash
npx hardhat compile
```

2. 启动本地节点，临时创建本地的EVM模拟环境，用于快速测试

```bash
npx hardhat node
```

3. 测试，可以运行单元测试

```bash
npx hardhat test
```

4. 本地运行

```bash
npx hardhat run scripts/deploy.js --network localhost
```

## 2.6 部署测试网络

在vscode安装`dotenv`插件，用于安全的管理私钥和API密钥

```bash
npm install dotenv
```

创建`.env`文件，<span class="r">不能上传到Git</span>

```
SEPOLIA_RPC_URL=YOUR_ALCHMEY_OR_INFURA_URL
PRIVATE_KEY=YOUR_METAMASK_PRIVATE_KEY
```

配置`hardhat.config.js`  修改配置文件以识别测试网络和私钥

```js
require("@nomicfoundation/hardhat-toolbox");
require("dotenv").config();

module.exports = {
  solidity: "0.8.20",
  networks: {
    sepolia: {
      url: `https://eth-sepolia.g.alchemy.com/v2/${process.env.API_KEY}`,
      accounts: [process.env.PRIVATE_KEY]
    },
    hoodi: {
      url: `https://hoodi.rpc.chainstack.com/${process.env.API_KEY}`,
      accounts: [process.env.PRIVATE_KEY]
    }
  }
};
```

然后在测试网络上部署
```bash
npx hardhat run scripts/deploy.js --network sepolia
```
