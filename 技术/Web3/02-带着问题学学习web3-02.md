
# 1. MetaMask钱包是联网的，那么我的私钥安全吗？

**核心答案：私钥的安全性取决于我们的设备**

在这里说明一下MetaMask钱包的工作原理
- MetaMask联网的部分**仅仅是连接到区块链网络来读取数据和广播交易**
- 其他的执行关键的操作都是本地化的

**A. 本地加密存储**
我们的私钥或者助记词是加密后**存储**在我们自己的浏览器的本地配置文件中的。

**B.签名在本地完成**
在我们要发送交易或者与智能合约交互的时:
- metaMask从设备内存中解密私钥
- 私钥仅用于对交易进行数字签名，不会进行上传私钥

# 2. NFT是不是伪去中心化？

绝大数的NFT都是伪去中心化，因为虽然在区块链上的信息以及资产的所有权确实无法篡改，但是这个资产是一个艺术品或者图片，有的人不会把这些存储在区块链中，因为成本不可控。

现在主流的玩法，是核心数据比如所有权记录 存在链上；具体的资产，比如图像、视频或者其他大文件，通常不会直接存储在链上成本极其昂贵，Gas费用爆炸，而且效率很低下。

那么，**资产定位符**指向的位置，应该如何存储呢？

**A.中心化存储**

这个也称为： 按位置寻址
这个很好理解，比如存在阿里云上OSS或者Amazon的S3上，这个很好理解
但是这个依然不安全，可以篡改这个文件的内容。，也许地址不变，但是内容已经发生了变化

**B.去中心化存储**

这个也称为：按照内容寻址

1. **IPFS    InterPlanetary File System**

它是高效的P2P文件分发系统，它不是区块链。

这个是目前最符合Web3精神的存储方式，tokenURI 通常用`ipfs://` 开头，， 链接到资产的内容hash
他们的地址是根据**文件内容本身计算出来的hash值** ；所以，如果文件内容哪怕只改变一个字节，它所对应的CID（地址）就会发生变化。

IPFS使用一个分布式hash表（DHT）存储哪些节点存储了CID信息，当我们请求CID的数据的时候，IPFS会向DHT查询哪些节点拥有该数据，然后直接从该节点中获取

*如何使用？*
- 上传：需要使用IPFS客户端或者付费服务商（比如Pinata、Infura）将文件添加到IPFS网络
- 持久性：==IPFS不保证永久存储==，如果拥有数据的节点决定删除或者下线，那么数据可能就消失了。因为，我们需要购买==固定服务（Pinning）==，支付费用让他们承诺永久保存文件（听着就不太可靠👀）

2. **Arweave  永久存储网络**

$Arweave$是一个建立在独特的数据块编织结构上的网络，旨在提供永久、一次性付费的存储服务。

- 数据块编织： 类似区块链，但它要求每一个新的数据块不仅要连接到上一个数据块，还要连接到一个随机的、过去的『召回数据块』
- 访问证明： Proof Of Access  (PoA) ，这个是Arweave的共识机制。矿工在打包新数据块的时候，必须证明他们可以访问到随机选择的历史数据块。
- 经济奖励：  这个肯定的，没钱谁干呢

*如何使用？*
- 上传： 我们需要使用Arweave钱包和AR代币
- 持久性：只要Arweave网络继续运行，数据理论上就会永久存在。
- 一次性预付：   一次付费能够覆盖未来100年的存储成本
