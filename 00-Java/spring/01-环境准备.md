# 4. 闭关修炼的计划

## 4.1 准备工作

- **环境** 将源码的开发环境准备
- **心态**  接受~~『忽略80%的非核心代码（比如日志、校验）』~~，只抓住最核心的**20%**的骨干逻辑

## 4.2 第一个阶段: IOC容器---spring框架的根基

#### 4.2.1 Day 1 容器启动流程和BeanDefinition

攻克：`AbstractionApplicationContext.refresh()`。 这个是Spring的灵魂，必须要掌握这12个方法
重点：`DefaultListableBeanFactory` （Spring的心脏）

一定要手绘一份`refresh()`流程图

#### 4.2.2 Bean的生命周期

重点： `doCreateBean` 和 `三级缓存和循环依赖`

写一个Setter循环依赖，打断点看它如何从二级缓存跳到一级缓存的。

#### 4.2.3 后置处理器 （这个是扩展点）

重点：`BeanFactoryPostProcessor` 与 `BeanPostProcessor`
彻底理解：`Aware`接口 和`InitializingBean`

## 4.3 第二个阶段：AOP与声明式事务

#### 4.3.1 AOP代理机制

找到AOP的入口，重点看看`AnnotationAawreAspectAutoProxyCreator`
以及：动态代理执行链路

#### 4.3.2 声明式事务原理

重点：`@Transactional`解析
攻克： 事务管理器和传播机制以及隔离机制

## 4.4 SpringMVC 和SpringBoot自动装配

#### 4.4.1 Spring MVC 交互流程

重点： `DispatcherServlet` 初始化 以及相关的九大组件的加载
弄懂 一个请求的完整生命周期，`doService` -> `doDispatch`

#### 4.4.2 SpringBoot启动与自动装配

弄懂： `SpringApplication.run()`与监听器机制
弄懂：自动装配原理 `@EnableAutoConfiguration`
