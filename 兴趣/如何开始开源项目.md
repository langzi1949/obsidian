
# 1. 总览

学习任何一个开项目，不能一上来就扣细节，每行代码都要学习，那肯定是不现实的。

核心阶段：

<span class="b" style="font-size:1.8rem">1. 先将项目跑起来</span>
- 没跑起来项目，一切都是扯淡
- 所以说当前阶段： 自己构建、启动、跑几个用例看看

<span class="b" style="font-size:1.8rem">2. 看项目整体目录结构</span>
- 看看项目的分层逻辑是啥

<span class="b" style="font-size:1.8rem">3. 画一些核心的架构图</span>

<span class="b" style="font-size:1.8rem">4. 从入口函数开始，查看核心流程</span>
- 在此阶段需要改代码，加日志等手段，同时依旧需要画流程图、时序图等

<span class="b" style="font-size:1.8rem">5. 有能力的话，修改代码，提PR</span>

总结一句话：
学开源项目源码的正确姿势是：
**先跑通 → 再建模 → 看主线 → 深模块 → 动手改。**

📍不要“乱点函数”，要“带着问题”去读。
📍不要急着背代码，要“画图”和“验证执行过程”。

## 📌 使用建议

1. **入门级**: 先运行起来，理解基本概念，修改小功能
2. **中级**: 阅读核心模块，理解架构设计，提交PR
3. **高级**: 深入性能优化，研究设计模式，解决复杂问题
4. **资深级**: 阅读底层实现，理解核心原理，引领技术方向

**难度说明**:
- ⭐ = 初学者友好
- ⭐⭐ = 需要基础知识
- ⭐⭐⭐ = 需要一定经验
- ⭐⭐⭐⭐ = 需要深厚积累
- ⭐⭐⭐⭐⭐ = 需要专家级理解
